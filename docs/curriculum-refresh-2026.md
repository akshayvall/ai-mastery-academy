# Curriculum Refresh 2026 - AI Mastery Academy

## Purpose
Upgrade the academy from prompt-centric learning to AI systems architecture learning while preserving existing module IDs and learner progress.

## Why This Refresh
The 2026 platform landscape shifted from single-model chat usage to production AI runtime design:
- Managed agents are now first-class platform primitives.
- MCP has moved from optional integration to core interoperability layer.
- Context engineering (caching, compaction, memory, routing) is now as important as prompting.
- Model deprecation velocity requires portfolio and migration literacy.
- Multimodal realtime systems are now mainstream product surfaces.

## Learning Outcomes (Foundational + Senior)

### Foundational outcomes
- Explain tokenization, context windows, retrieval, tool calls, and structured outputs.
- Build and debug single-agent and multi-tool workflows safely.
- Operate model APIs with correct key management, schema contracts, and cost controls.

### Senior outcomes
- Design model routing strategies by task class, cost, latency, and reliability.
- Architect agent systems with approval boundaries, side-effect controls, and observability.
- Manage migration and deprecation risk across OpenAI, Anthropic, and Google model ecosystems.

## Syllabus updates in this pass

### Level 100
1. New flagship module: 2026 AI systems landscape.
2. Phase 3 reframed around model lifecycle, routing, and production API operations.

### Level 200
1. Phase 4 reframed as context engineering plus RAG.
2. Phase 5 reframed as context engineering plus tool orchestration.
3. Phase 6 reframed around managed agent loops and approval boundaries.
4. Phase 7 updated to include managed agents and Claude platform operations.
5. Phase 8 updated to include orchestrator-worker plus hosted multiagent sessions.

## Recommended next pass (not yet implemented)
1. Add a dedicated model routing lab with workload traces and routing policies.
2. Add a multimodal realtime module (speech, live sessions, interruption handling, latency budgets).
3. Add a migration engineering module for model deprecations and contract compatibility testing.
4. Add an evals and reliability module focused on tool/agent traces and regression gates.

## Design constraints
- Keep existing phase IDs to preserve localStorage progress.
- Favor additive content updates over deep structural refactors in this pass.
- Keep the site static (HTML/CSS/JS only) with no build step.
